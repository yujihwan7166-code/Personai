## 🔴 최우선 원칙 (절대 어기지 말 것)

1. **완성도 우선** — 기능이 많은 것보다 있는 기능이 완벽하게 작동해야 함
2. **실제 사용 가능** — "구현은 됐지만 어색한" 상태는 완성이 아님
3. **단계 승인 필수** — 각 Phase 완료 시 보고하고 승인 받은 후 다음 단계로
4. **추측 금지** — 불확실한 건 진행 전에 반드시 먼저 질문
5. **빌드 체크 의무** — 각 Phase 끝날 때마다 빌드 에러 없는지 확인

## ⛔ 하지 말 것 (항상 적용)

- 계획 없이 코드 짜기 시작
- 에러 무시하고 다음 단계 진행
- `any` 타입 사용
- 관련 없는 파일 수정
- 라이브러리 무단 추가 (먼저 물어볼 것)
- "일단 동작하면 됨" 수준으로 마무리
- 로딩/에러/빈 상태 중 하나라도 빠뜨리기
- 모바일 확인 생략
