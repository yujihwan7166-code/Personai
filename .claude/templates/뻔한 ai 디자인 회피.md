# Anti-AI-Slop 멀티 에이전트 반복 템플릿
## AI가 만든 티 안 나는 디자인을 만드는 자동 검증 시스템

---

## 이 템플릿이 해결하는 문제

AI에게 디자인을 시키면 전 세계 어디서나 같은 결과가 나옵니다.
보라색 그라디언트, Inter 폰트, 둥근 카드, 글래스모피즘, 모호한 헤드라인.
이걸 업계에서 **"AI Slop"**이라고 부릅니다.

이 템플릿은 기획자→생성자→평가자가 반복 사이클을 돌면서
AI Slop 패턴을 자동 탐지하고 제거합니다.
평가자에게 "AI Slop 체크리스트"를 내장시켜서
한 항목이라도 걸리면 자동 반려됩니다.

---

## 사용법

1. 아래 프롬프트를 통째로 AI에게 입력
2. 🔧 표시된 변수만 본인 프로젝트에 맞게 수정
3. AI가 기획→생성→평가를 순차 실행
4. 평가자가 AI Slop 패턴을 발견하면 자동 반려 → 재생성
5. 모든 기준 통과할 때까지 반복

---

## 타이포그래피 블랙리스트

## 레이아웃 블랙리스트
- 3열 카드 그리드 + 아이콘 + 설명 텍스트 (가장 흔한 AI 패턴)
- 히어로 → 3열 피처 → 사용후기 → 가격 → CTA (SaaS 템플릿 구조)
- 모든 카드가 동일한 border-radius (보통 12~16px)
- 모든 요소에 동일한 padding/gap
- 박스 안에 박스 안에 박스 (중첩 카드)

## 시각 효과 블랙리스트
- 모든 카드에 동일한 box-shadow
- backdrop-filter: blur()로 만든 글래스모피즘 (2024년에 이미 사망)
- 의미 없는 블롭/원형 그라디언트 배경 장식
- 얼굴 없는 3D 캐릭터 일러스트

## 생성 원칙 (모든 스프린트에 적용)

### 원칙 1: 의도적 불균형
- 모든 요소를 균등하게 배치하지 마세요 (AI의 기본 습관)
- 한 요소를 의도적으로 크게, 다른 건 작게
- 여백을 비대칭으로 사용
- 그리드를 깨는 요소를 최소 1개 포함

### 원칙 2: 질감과 깊이
- 단색 배경 금지. 미세한 노이즈, 텍스처, 패턴을 레이어링
- 그림자는 요소마다 다른 강도와 방향으로 (동일 shadow 금지)
- 배경에 의미 있는 시각 요소 (단순 블롭/원형 장식 금지)

### 원칙 3: 타이포 위계의 과감함
- 제목과 본문의 크기 차이를 극적으로 (최소 3:1 비율)
- font-weight 변화를 적극적으로 사용
- 자간(letter-spacing)으로 분위기 조절
- 한 화면에 최소 2가지 다른 타이포 스타일

### 원칙 4: 컬러의 절제와 강조
- 전체 팔레트를 3색 이하로 제한 (AI는 5~6색을 균등하게 쓰려 함)
- 주 색상 1개를 압도적으로 사용, 나머지는 포인트
- 같은 색의 opacity 변형으로 차이를 만들지 마세요 (AI 습관)
- 대신 명도/채도가 다른 별도 색상을 사용

### 원칙 5: 애니메이션의 의미
- "있으니까 넣는" 애니메이션 금지
- 각 애니메이션에 "왜 움직이는지" 이유가 있어야 함
- 모든 요소가 같은 타이밍으로 나타나지 않도록 stagger 필수
- easing은 ease-out 일변도 금지. 맥락에 맞는 커브 선택

## 1단계: AI Slop 탐지 (통과/실패, 1개라도 걸리면 즉시 반려)

아래 체크리스트를 모든 스프린트에 적용하세요.

### 색상 체크
□ 보라색→파란색 그라디언트를 사용하지 않았는가?
□ Tailwind 기본 indigo/violet/blue 계열을 피했는가?
□ 파스텔 3색 조합(라벤더/피치/민트)이 아닌가?
□ 색상 선택에 프로젝트 맥락에서 비롯된 이유가 있는가?
□ 같은 색의 opacity 변형만으로 팔레트를 구성하지 않았는가?

### 타이포 체크
□ Inter, Roboto, Poppins, DM Sans, Space Grotesk를 쓰지 않았는가?
□ 폰트 선택에 구체적인 이유가 있는가?
□ 제목과 본문의 크기/무게 차이가 충분한가?
□ 전체적으로 2가지 이상의 타이포 스타일이 공존하는가?

### 레이아웃 체크
□ 3열 균등 카드 그리드가 주요 구조가 아닌가?
□ 모든 카드/요소의 border-radius가 동일하지 않은가?
□ 비대칭 요소가 최소 1개 존재하는가?
□ 여백(padding/margin)이 요소마다 의도적으로 다른가?
□ "박스 안의 박스" 중첩이 2단계를 넘지 않는가?

### 시각 효과 체크
□ 모든 카드에 동일한 box-shadow를 쓰지 않았는가?
□ backdrop-filter: blur() 글래스모피즘에 의존하지 않는가?
□ 의미 없는 원형/블롭 배경 장식이 없는가?
□ 질감이나 텍스처가 최소 1개 적용되었는가?

### 카피 체크 (해당 시)
□ "Build the future" 류의 모호한 헤드라인이 없는가?
□ 카피가 제품/서비스의 구체적 가치를 전달하는가?
