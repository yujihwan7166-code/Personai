# 멀티 에이전트 프롬프트 — 범용 템플릿

<사용법>
지금부터 (하고자 하는것을 구체적으로 작성) 할꺼야.
기획-생성-평가 구조 프롬프트를 보고 총 20번 반복해서 최선의 품질을 제공할 수 있는 구현계획 세워봐





## 이 문서가 뭔가요?

AI한테 복잡한 작업을 시킬 때,
하나의 AI가 **기획 → 생성 → 평가**를 순서대로 돌면서
스스로 품질을 검증하고 개선하는 구조입니다.

아래 템플릿에서 🔧 표시된 부분만 본인 프로젝트에 맞게 바꾸면 됩니다.

---

## 적용 예시 (이런 곳에 다 쓸 수 있음)

| 프로젝트 | 기획자가 할 일 | 생성자가 할 일 | 평가자가 볼 것 |
|----------|---------------|---------------|---------------|
| 웹사이트 디자인 | 페이지 구조, 사용자 동선 정의 | UI 디자인, 코드 구현 | 시각 품질, UX, 반응형 |
| 사업계획서 | 시장분석, BM 설계 | 문서 작성, 도표 생성 | 논리성, 실현 가능성, 설득력 |
| 마케팅 전략 | 타겟 정의, 채널 선택 | 카피라이팅, 콘텐츠 설계 | 전환율 예측, 톤앤매너, 차별성 |
| 앱 기획 | 기능 목록, 화면 흐름 | 와이어프레임, 프로토타입 | 사용성, 기술 실현성, 수익 모델 |
| 게임 설계 | 게임 메카닉, 보상 루프 | 레벨 디자인, UI 구현 | 재미, 밸런스, 리텐션 |
| 유튜브 채널 | 콘텐츠 전략, 시리즈 구성 | 대본 작성, 썸네일 기획 | 클릭률 예측, 시청 유지율, 독창성 |
| 이력서/포폴 | 강점 분석, 구조 설계 | 문장 작성, 레이아웃 | 임팩트, 가독성, ATS 통과율 |

## 평가 출력 형식

각 스프린트별로:
1. 점수표 (5개 기준 × 점수)
2. 강점 2가지 (구체적으로)
3. 약점 2가지 (구체적으로)
4. 개선 제안 (약점을 어떻게 고칠지 구체적 방법)

종합 평가:
- 전체 평균 점수
- 가장 시급하게 개선할 영역 1개와 그 이유
- "만약 한 가지만 바꿀 수 있다면" — 가장 임팩트 큰 변경 1개

## 고급 옵션 (선택사항)

### 품질 강제 올리기
프롬프트 마지막에 추가:
"평가자는 첫 번째 사이클에서 반드시 1개 이상의 스프린트를 반려하세요.
완벽해 보여도 개선점을 찾아 반려하고, 두 번째 사이클에서 통과 여부를 판단하세요."

### 에이전트 4개로 확장
생성자와 평가자 사이에 [테스터]를 추가할 수 있습니다:
"[에이전트 2.5: 테스터] — 생성자의 코드를 실제로 실행하는 시나리오를 작성하고,
엣지 케이스 5가지를 시뮬레이션한 결과를 보고하세요."

### 경쟁 구조 (같은 역할 2개)
생성자를 2명 두고 서로 다른 방향으로 만들게 한 뒤,
평가자가 둘 중 더 나은 걸 고르게 할 수 있습니다:
"[생성자 A]는 미니멀한 방향으로, [생성자 B]는 화려한 방향으로 작업하세요.
[평가자]는 둘을 같은 기준으로 채점하고, 더 높은 쪽을 선택하세요."
